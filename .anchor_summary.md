## Goal
Transform the super admin dashboard into a fully operational command center with sidebar navigation, live KPIs, charts, filtered tables, per‑admin product assignment, and user lifecycle tracking — accessible via PIN gate without GoRouter auth redirects.

## Constraints & Preferences
- All PINs must be strictly 4 digits (enforced everywhere: set_pin, add/edit employee, add user, login)
- Super admin credentials (PIN 0000) must not be visible on the login screen
- The POS icon on the login page must require PIN 8667 before accessing operations
- Account lifecycle: admin created → default PIN 1234 → must change PIN and set name → becomes "active" member visible on super admin page
- Exports and tables should follow the admin pages pattern (DataTable with horizontal scroll, CSV/PDF via showExportDialog)
- Hardcoded PINs: super admin = 0000, default admin = 1234, operations access = 8667
- Navigation inside super admin must bypass GoRouter auth redirect (use Navigator.push with MaterialPageRoute)
- DDC enums can become null during hot reload — use int constants instead of enum values in state
- Responsive breakpoints: < 600px mobile (bottom nav), 600–899px tablet (mini sidebar), ≥ 900px desktop (full sidebar)
- New `String? assignedTo` field on Product and InventoryTransaction tracks which admin is responsible; nullable default = unassigned
- Avoid manual Product(...) construction — use `copyWith(...)` everywhere to preserve `assignedTo` automatically

## Key Decisions
- Use direct HiveDatabase reads in super admin dashboard (no BLoC layer) for simplicity and immediate data access — matches existing operations_page.dart pattern
- Merge operations monitoring into super admin dashboard rather than keeping separate pages
- Navigation via Navigator.push instead of GoRouter inside PIN-gated pages — avoids GoRouter auth redirect while PIN gate (8667) provides authorization
- DDC hot reload safety: int constants instead of enum for section state to avoid null enum values
- Responsive layout via single LayoutBuilder instead of separate route trees — full sidebar ≥ 900px, mini sidebar 600–899px, bottom nav < 600px
- Analytics kept as inline section (not a separate page) to preserve sidebar continuity
- `assignedTo` on Product is a simple nullable HiveField — no separate join table or box; null means unassigned; old data migrates gracefully
- `copyWith()` added to Product (and enhanced on InventoryTransaction) to make adding new fields in the future less error‑prone
- Zero-value PieChart sections filtered out to avoid fl_chart web rendering crash; TextStyle objects in chart tooltips not const to prevent JS canonicalization issues
